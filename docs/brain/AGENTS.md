## Ⅰ. 工程化闭环 (Architecture & Workflow)

The system operates on a strict, 5-step lifecycle:

1. 🏗️ **物理挂载 (Rebuild)**: Reconstructs the ephemeral operational memory directly from the immutable historical ledgers.
2. 📡 **双重防抖感知 (Harvest)**: Securely fetches external data with multiple physical validation layers, actively blocking false updates or redundant noise.
3. 👁️ **全知降维吞噬 (Ingest)**: Dissects and structurally maps the internal codebase into deterministic configuration nodes, without relying on massive external parsers.
4. 🤔 **纯数学图论推演 (Ponder)**: Evaluates the system's structural integrity, identifies orphaned concepts, and securely generates prioritized operational targets.
5. 🛡️ **沙盒免疫合入 (Evolve)**: Verifies the systemic logic flow in a secure sandbox. The pipeline halts instantly if any verification fails.

All Agents (including Jules) must prioritize: **Zero-Dependency Brutalism, Append-Only Memory, and Machine Draft, Human Decision**.

## Ⅱ. 核心法则 (Core Philosophy)

1. **零依赖极简 (Zero-Dependency Brutalism)**: Rejects bloated external frameworks. Relies entirely on highly stable, native components.
2. **不可篡改 (Append-Only Memory)**: History is immutable. Architectural modifications or logic corrections are handled strictly via safe deprecation edges and temporal versioning.
3. **算力主权 (Machine Draft, Human Decision)**: Automated systems propose structural updates; human architects execute the final integration and verification.
